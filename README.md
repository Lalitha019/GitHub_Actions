# GitHub Actions CI Pipeline

## Overview

This project demonstrates how to implement a Continuous Integration (CI) pipeline using GitHub Actions. The workflow automatically triggers on every code push, installs dependencies, and executes Python tests to validate the application.

## Features

* Automated workflow execution on code push
* Python environment setup using GitHub Actions
* Dependency installation using pip
* Automated test execution using Pytest
* Matrix build strategy for multiple Python versions
* Hands-on experience with GitHub Hosted and Self-Hosted Runners

## Tech Stack

* GitHub Actions
* Python
* Pytest
* YAML
* CI/CD

## Workflow

1. Developer pushes code to GitHub.
2. GitHub Actions workflow is triggered automatically.
3. Python environment is created.
4. Required dependencies are installed.
5. Test cases are executed.
6. Build results are generated and displayed.

## Key Learnings

* CI/CD Fundamentals
* GitHub Actions Workflows
* Matrix Build Strategy
* Workflow Automation
* Runner Configuration
* Automated Testing

## Challenges Faced

* Resolved workflow configuration and YAML syntax issues.
* Troubleshooted test execution and runner setup challenges.
* Validated application compatibility across multiple Python versions.

## Outcome

Successfully built and executed an automated CI pipeline that validates code changes using GitHub Actions, improving code quality and reducing manual testing effort.
